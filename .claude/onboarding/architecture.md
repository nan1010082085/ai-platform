## 状态管理架构

### 1. Store 层次结构

```
useAiStore (主 Store)
    │
    ├── useConversationStore (对话管理)
    │   ├── conversations: Conversation[]
    │   ├── currentConversationId: string
    │   └── messages: AIMessage[]
    │
    ├── useStreamStore (流式连接)
    │   ├── loading: boolean
    │   ├── error: string
    │   └── streamStatus: SSEStatus
    │
    ├── useSchemaStore (Schema 状态)
    │   ├── currentSchema: Widget[]
    │   ├── currentFlow: FlowGraph
    │   └── schemaHistory: SchemaHistory[]
    │
    ├── useLLMStore (LLM 管理)
    │   ├── providers: LLMProvider[]
    │   └── currentProvider: string
    │
    ├── useRAGStore (RAG 搜索)
    │   ├── searchResults: SearchResult[]
    │   └── context: RagContext[]
    │
    ├── useChatSettingsStore (聊天设置)
    │   └── settings: ChatSettings
    │
    └── useHITLStore (Human-in-the-loop)
        └── pendingInterrupt: Interrupt
```

### 2. 数据流向

```
用户输入
    │
    ▼
useAiStore.sendMessage()
    │
    ├── 追加用户消息到 conversationStore.messages
    │
    ├── 创建 assistant 消息占位
    │
    └── 调用 streamStore.executeStream()
            │
            ├── 建立 SSE 连接
            │
            ├── 接收流式事件
            │
            └── 调用 handleStreamEvent()
                    │
                    ├── 更新消息内容
                    │
                    ├── 更新 Schema 状态
                    │
                    └── 更新工具调用状态
```
