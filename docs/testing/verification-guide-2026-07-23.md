## 一、Workflow 新节点（批次1）

### 1.1 代码执行节点（code-execute）

**前置**：创建工作流，拖入 code-execute 节点

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 节点面板选择语言 "JavaScript" | 下拉只显示 JavaScript |
| 2 | 代码输入 `return { result: $input.message }` | 文本框正常显示 |
| 3 | 连接 trigger → code-execute → end，点"调试" | WorkflowDebugView 打开 |
| 4 | 输入"测试消息"，点执行 | 节点轨迹显示 code-execute 成功，输出 `{ result: "测试消息" }` |
| 5 | 输入错误代码 `throw new Error('fail')` 执行 | 节点标红，错误信息"代码执行失败: Error: fail" |
| 6 | 输入空代码执行 | 校验报错"代码脚本为空" |

### 1.2 变量赋值节点（variable-set）

**前置**：创建工作流，拖入 variable-set 节点

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 变量名 `myVar`，值 `{{$input.message}}`，模式 `set` | 配置正常保存 |
| 2 | 连接 trigger → variable-set → LLM（prompt 引用 `$node.__variables.myVar`）→ end |  |
| 3 | 输入"hello"，执行 | variable-set 输出 `{ variable: "myVar", value: "hello", mode: "set" }` |
| 4 | 模式改 `increment`，值 `10`，执行两次 | 第一次输出 10，第二次输出 20 |
| 5 | 模式改 `append`，值 `{{$input.message}}`，执行两次 | 变量变成数组 `["msg1", "msg2"]` |
| 6 | 变量名留空执行 | 校验报错"变量赋值节点变量名为空" |

### 1.3 多路分支节点（switch）

**前置**：创建工作流，拖入 switch 节点

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 默认 2 个分支：A=`true`, B=`false` | 面板显示两个分支输入 |
| 2 | 点"添加分支"加第三个 | 出现第三个分支输入 |
| 3 | 删第二个分支 | 只剩两个分支 |
| 4 | 连接 trigger → switch → 两个 end，switch 出两条边 |  |
| 5 | 执行，expression 匹配 A=true | 输出 `{ branch: "分支A", index: 0 }` |
| 6 | 执行，expression 都不匹配 | 输出 `{ branch: "default", index: -1 }` |
